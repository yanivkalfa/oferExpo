# OfersPhone — Voice-to-Text Mobile App

A simple Android/iOS app: tap **Record**, speak, see your words as text. Supports
Hebrew, English, Russian, Arabic and more, with a language picker.

Built with **Expo (React Native)** + **expo-speech-recognition**.

## What you get

- TextInput showing live (interim + final) transcript as you speak.
- **Record / Stop** button.
- **Clear** button.
- Horizontal language chips: Auto, עברית, English, Русский, العربية.
- Default language is `he-IL` (Hebrew). Tap another chip to switch before recording.

## Prerequisites (install these first)

You can either install everything fresh, or — if you already have **Unity 6000.4.x**
with Android build support — reuse Unity's bundled JDK + Android SDK + NDK
(see *Reusing Unity's Android toolchain* below).

### 1. Node.js 25.x (or 20.x LTS)

Windows (winget):
```powershell
winget install OpenJS.NodeJS
```
Verify:
```powershell
node -v   # v25.x or v20.x
npm -v
```

### 2. Git

Windows (winget):
```powershell
winget install Git.Git
```

### 3. JDK 17

Either install Eclipse Temurin 17:
```powershell
winget install EclipseAdoptium.Temurin.17.JDK
```
…or reuse Unity's bundled JDK at:
```
C:\Program Files\Unity\Hub\Editor\<UnityVersion>\Editor\Data\PlaybackEngines\AndroidPlayer\OpenJDK
```

### 4. Android SDK + NDK

Easiest: install **Android Studio**, open SDK Manager, install:
- **Android SDK Platform 36** (API 36)
- **Android SDK Build-Tools 36.0.0**
- **Android SDK Command-line Tools (latest)**
- **Android SDK Platform-Tools** (gives you `adb`)
- **NDK 27.1.12297006** (or newer 27.x)
- **CMake 3.22.1**

Windows (winget):
```powershell
winget install Google.AndroidStudio
```

### Reusing Unity's Android toolchain (no Android Studio needed)

If Unity is installed with Android build support, all of the above already exist
under:
```
C:\Program Files\Unity\Hub\Editor\<UnityVersion>\Editor\Data\PlaybackEngines\AndroidPlayer\
   OpenJDK\         <- JDK 17
   SDK\             <- Android SDK (platforms, build-tools, platform-tools, cmdline-tools)
   SDK\ndk\27.1.12297006\
   SDK\cmake\3.22.1\
```

> ⚠️ **Important — short-path linker bug.**
> Building C++ from a path containing spaces like `C:\Program Files\...` causes
> the NDK linker to use the 8.3 short name `CLANG_~1.EXE`, which then fails to
> resolve libc++ symbols (you'll see hundreds of `undefined symbol: std::__ndk1::...`
> errors).
>
> Fix: create a junction at a short, space-free path:
> ```powershell
> # Run PowerShell as Administrator
> cmd /c mklink /J C:\Android\sdk "C:\Program Files\Unity\Hub\Editor\<UnityVersion>\Editor\Data\PlaybackEngines\AndroidPlayer\SDK"
> ```
> Then point `ANDROID_HOME` at `C:\Android\sdk` (see env vars below).

## Configure environment variables

Set these as **User** environment variables in Windows (one time, persistent):

```powershell
# Adjust the JDK path if you didn't use Unity's bundled OpenJDK.
[Environment]::SetEnvironmentVariable(
  'JAVA_HOME',
  'C:\Program Files\Unity\Hub\Editor\6000.4.6f1\Editor\Data\PlaybackEngines\AndroidPlayer\OpenJDK',
  'User')

[Environment]::SetEnvironmentVariable('ANDROID_HOME',     'C:\Android\sdk', 'User')
[Environment]::SetEnvironmentVariable('ANDROID_SDK_ROOT', 'C:\Android\sdk', 'User')

# Append to PATH (only do this once)
$user  = [Environment]::GetEnvironmentVariable('Path','User')
$extra = 'C:\Android\sdk\platform-tools;C:\Android\sdk\cmdline-tools\latest\bin;C:\Android\sdk\emulator'
[Environment]::SetEnvironmentVariable('Path', "$user;$extra", 'User')
```

Close and reopen PowerShell. Verify:
```powershell
java -version           # 17.x
adb --version
echo $env:JAVA_HOME
echo $env:ANDROID_HOME
```

If PowerShell blocks scripts (npm/npx), allow per-user scripts (one time):
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## Run the app on a real Android phone (debug)

1. On the phone: **Settings → About phone**, tap **Build number** 7 times to
   enable Developer Options. Then **Settings → System → Developer options →
   Enable USB debugging**.
2. Plug the phone into the PC, accept the "Allow USB debugging" prompt on the phone.
3. Verify the PC sees the phone:
   ```powershell
   adb devices
   # should list your phone with status "device" (not "unauthorized")
   ```
4. Build, install, and launch (~3–5 min the first time, ~30 sec later):
   ```powershell
   cd c:\path\to\oferExpo
   npx expo run:android
   ```

   If your phone is **arm64** only (most modern phones), the included
   `android/gradle.properties` already restricts to `arm64-v8a` to halve build
   time. To build all ABIs, edit `reactNativeArchitectures=arm64-v8a` to
   `arm64-v8a,armeabi-v7a,x86,x86_64`.

5. Once the app is on the phone, edits to `App.tsx` hot-reload via Metro
   (the dev server window). Press `r` in the Metro terminal to force reload,
   or shake the phone for the dev menu.

### iOS

`npx expo run:ios` — requires macOS + Xcode. Not covered here.

## Build a signed release APK

This produces a standalone APK that runs without Metro and can be shared.

### 1. Create a release keystore (one time)

> ⚠️ **Back this file up.** If you lose it you can never publish updates that
> upgrade an existing install — users would have to uninstall + reinstall.

```powershell
cd c:\path\to\oferExpo\android\app
& "$env:JAVA_HOME\bin\keytool.exe" -genkeypair -v -storetype PKCS12 `
  -keystore ofersphone-release.keystore -alias ofersphone `
  -keyalg RSA -keysize 2048 -validity 10000 `
  -dname "CN=OfersPhone, OU=Dev, O=Yaniv, L=Unknown, ST=Unknown, C=IL" `
  -storepass <YOUR_STORE_PASSWORD> -keypass <YOUR_KEY_PASSWORD>
```

`android/app/build.gradle` is already wired to use this keystore in the
`release` build type. If you change the filename / passwords / alias, edit the
`signingConfigs.release { ... }` block accordingly.

### 2. Build the APK

```powershell
cd c:\path\to\oferExpo\android
.\gradlew.bat :app:assembleRelease -PreactNativeArchitectures=arm64-v8a
```

Output:
```
android\app\build\outputs\apk\release\app-release.apk
```

### 3. Install on your phone

```powershell
adb install -r android\app\build\outputs\apk\release\app-release.apk
```

Or copy the APK file to any phone, tap it, allow "install unknown sources".

### Bonus: Google Play Bundle

```powershell
.\gradlew.bat :app:bundleRelease -PreactNativeArchitectures=arm64-v8a
# output: android\app\build\outputs\bundle\release\app-release.aab
```
Upload the `.aab` to the Google Play Console.

## Common errors & fixes

### `JAVA_HOME is not set and no 'java' command could be found`
Your PowerShell session predates the env-var setup. Open a new PowerShell, or
load them into the current one:
```powershell
$env:JAVA_HOME = [Environment]::GetEnvironmentVariable('JAVA_HOME','User')
$env:ANDROID_HOME = [Environment]::GetEnvironmentVariable('ANDROID_HOME','User')
$env:Path = "$env:JAVA_HOME\bin;$env:ANDROID_HOME\platform-tools;" + $env:Path
```

### `Failed to resolve the Android SDK path. Default install location not found`
`ANDROID_HOME` not set. See env-vars section.

### `ld.lld: error: undefined symbol: std::__ndk1::basic_string<...>` (hundreds of these)
8.3 short-path bug. Your SDK/NDK lives under `C:\Program Files\...`.
Create the `C:\Android\sdk` junction and update `ANDROID_HOME` (see
*Reusing Unity's Android toolchain*). Then:
```powershell
Remove-Item .\android\app\.cxx -Recurse -Force
Remove-Item .\node_modules\expo-modules-core\android\.cxx -Recurse -Force
npx expo run:android
```

`android/local.properties` in this repo already pins `sdk.dir` and `ndk.dir`
to `C:\Android\sdk` to avoid this. Edit it if your junction is elsewhere.

### Speech error: `audio-capture: No service found for package com.google.android.googlequicksearchbox`
Your phone doesn't have the Google app installed. Either:
- Install the **Google** app from the Play Store, or
- Don't pin `androidRecognitionServicePackage` (already removed in `App.tsx`).

### Hebrew speech comes out as English-letter gibberish
You're using the **Auto** chip, but Android's recognizer doesn't actually
auto-detect language unless the Google recognition service is installed and
the device honors `EXTRA_ENABLE_LANGUAGE_DETECTION`.
**Fix:** tap the **עברית** chip before recording. Default is already `he-IL`.

### `Setting newArchEnabled=false ... is not supported anymore since React Native 0.82`
Don't disable the new architecture. RN 0.82+ requires TurboModules; that's
why we use `expo-speech-recognition` (which supports them) and not the
unmaintained `@react-native-voice/voice`.

### Gradle daemons hang or "appears stuck"
Stop them:
```powershell
cd android
.\gradlew.bat --stop
```

## Project structure

```
oferExpo/
├── App.tsx              # All UI + speech recognition logic
├── app.json             # Expo config (permissions, plugin, package id)
├── eas.json             # Optional: EAS Build profiles (cloud builds)
├── package.json
├── tsconfig.json
├── assets/              # Icons + splash
├── index.ts             # Entry (registers App)
├── android/             # Generated by `expo prebuild` — gitignored
└── ios/                 # Generated by `expo prebuild` — gitignored
```

The `/android` folder is regenerated automatically; don't be surprised if it's
missing after `git clone` — `npx expo run:android` will create it.

If you need to keep custom native edits, commit the relevant files manually
or add an Expo config plugin.

## Phone-side setup for Hebrew

For best Hebrew (and offline) recognition:

1. Install the **Google** app from Play Store (provides
   `com.google.android.googlequicksearchbox` recognition service).
2. Open the Google app → profile → **Settings → Voice → Offline speech
   recognition** → download **Hebrew (Israel)**.
3. Set device locale to Hebrew if you want Auto mode to default to Hebrew.

For other languages, repeat step 2 with the languages you need.
